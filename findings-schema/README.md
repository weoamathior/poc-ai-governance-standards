# Findings Schema

This directory defines the structure of a single evaluation finding. The schema lives in
`finding.schema.json` and is written as JSON Schema (draft-07).

A finding is the unit of output the evaluator produces when it identifies a violation of a
standard. Every finding generated by the pipeline must conform to this schema. The schema
is not advisory documentation; it is the contract that keeps findings machine-readable and
consistent across every evaluation.

Conformance matters because findings are durable. At merge time, the findings produced for
a pull request are committed to the audit log of the target repository, where they form a
permanent forensic record of what the pipeline observed and how each observation was
ultimately handled. Because that record must remain readable and queryable long after the
pull request itself has faded from memory, every finding must have a predictable shape. The
schema is how that predictability is guaranteed.
